What is this report?

This view is not stored in the database. It is generated by combining data from multiple tables.

  • Trains — what train is
  • Bookings — who is book the train
  • Reservations — how trains and bookings are linked

The system dynamically joins these tables using SQL JOIN operations to produce a meaningful report.

SQL used to generate this report (JOIN)

Read-only 
SELECT
    r.id AS reservation_id,
    t.train_name AS train_name,
    b.passenger AS booking_passenger_name,
    r.reserved_on
FROM reservations r
JOIN trains t ON r.train_id = t.id
JOIN bookings b ON r.booking_id = b.id;
Note: Course information is not duplicated in the train table. This follows database normalization principles.
